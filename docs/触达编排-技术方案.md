# AI 招聘触达编排 — 技术方案

> 基于对秒回(MiaoHui)、秒懂(MiaoDong/Insight)完整能力的调研。
> 需求：HR 表格填好约面时间 → 表格服务发请求 → 触达服务编排：加好友 → 好友通过发欢迎语(带约面时间) → 候选人回复意图识别 → 分情况同步 HR → 全程回填表格；触达失败通知 HR。

---

## 一、能力边界（调研地面真相）

### 秒回（IM 手脚，只提供原子能力）
- **加好友**：`POST /addFriend/send`(小组级)，`helloMsg`申请话术，`extraInfo`透传 dataId(同号首次固定)，`userId`指定 HR 托管账号，`instant:true`。异步任务。
- **发消息**：`POST /message/send`(chatId 寻址，`externalRequestId` 幂等)。
- **回调**(全配在小组控制台→API接入，单一 URL)：好友通过 `/friend/confirm`、加好友结果 `/friend/send`(errorCode)、候选人消息 `/message`、发送结果 `/sentResult`、会话事件 `/chat`。
- **会话 AI 开关**：`/chat/setAiWorking`(转 AI/转人工，编排用)。
- ⚠️ **SOP/AISOP 无对外 API**，不能程序触发 → **触达服务自己是编排引擎**。
- ⚠️ 好友通过回调只给 wxid/externalUserId，发消息前要换 chatId(`getChatId`)。
- ⚠️ 回调必须 **200+异步+幂等**，否则地址被禁 30 分钟丢事件。

### 秒懂（AI 大脑，画布能包办对话全链路）
- **触发节点**：`wecom-contact-bind`(**企微好友通过**，输出 phonenum + isFirstTrigger)、`receive-text-message`(**候选人回复**)、`canvas-event-trigger`(定时/延迟)。
- **意图识别**：`llm-completion`(jsonOutput 分类) + `rule-center`(按意图分流)。
- **知识库答疑**：`query-knowledge-base`(FAQ 向量检索，输出 highestFaq)。
- **回报外部**：`plugin-calculation`(有返回值)/`plugin-action`(fire-forget) → **直连触达服务 HTTP，不经 MQ，90s 超时**。
- **发消息/转人工**：`send-text-message`(template 支持 {{变量}})、`handover`。
- **定时**：`canvas-event-action`(TIMER/SCHEDULE, delaySeconds/timerAfterDays) — 约面前一天提醒可用。
- **绑定**：托管账号 → 绑定智能体(bot.enabledCanvasId=招聘画布) → 开 AI 自动回复。候选人消息经 mh-ai-connector 自动进画布(triggerSource=MH)。

## 二、架构决策：**画布主导 + 触达服务做编排后端**

符合既定方向"秒懂画布管收发编排、服务只做逻辑"，且候选人回复经秒回联动**直接进画布**（不用触达服务中转）。

```
┌─表格服务─┐   ┌──────────── 触达服务(编排大脑) ────────────┐   ┌─秒回─┐   ┌─秒懂画布─┐   ┌─飞书应用─┐
│HR填约面时间│──▶│ 状态机(Mongo) + 加好友 + HR交互 + 回填 + 兜底 │◀─▶│加好友│   │好友通过→ │   │回填表 通知│
└──────────┘   └────────────────────────────────────────────┘   │发消息│   │欢迎语→接待│   │HR        │
                          ▲ plugin回报        ▲回调                │回调  │   │意图→答疑 │   └──────────┘
                          └───────────────────┴────秒懂画布────────┴──────┴──▶ plugin回报触达服务
```

## 三、完整时序

```
① 表格服务 ──POST /reach {dataId, phone, interviewTime, name?, position?}──▶ 触达服务
② 触达服务: Mongo 存触达任务(状态=ADDING, 关联 dataId↔phone↔约面信息)
            → 调秒回 /addFriend/send (extraInfo=taskId, userId=HR托管号, helloMsg=申请语)
③ 秒回加好友:
     ├ 失败 ──/friend/send(code=1,errorCode)──▶ 触达服务: 状态=ADD_FAILED → 飞书通知HR + 回填(触达失败)
     └ 好友通过:
         ├─ 秒回 ──/friend/confirm──▶ 触达服务: 状态=CONFIRMED, 存 wxid/externalUserId
         └─ 秒懂画布 wecom-contact-bind 触发(并行):
              → plugin 调触达服务 GET /logic/candidate-info?phone= (查约面信息)
              → send-text-message 发欢迎语(带 {{约面时间}}) → 状态=WELCOMED
④ 候选人回复 ──秒回联动──▶ 画布 receive-text-message:
              → llm-completion(意图分类: 确认/改约/拒绝/提问/其它)
              → rule-center 分流:
                 ├ 提问 → query-knowledge-base 答疑 → send 回复(不打扰HR)
                 ├ 确认/改约/拒绝 → plugin-calculation 调触达服务 /logic/report-intent
                 │                 → send 回复候选人(确认话术/问时间/收尾)
                 └ 复杂/要真人 → handover 转人工
⑤ 触达服务收 /logic/report-intent: 状态=REPLIED_{intent}
              → 飞书同步HR(候选人已确认X时间/要改约/拒绝) + 回填表格
⑥ 全程每次状态变化回填进度表; 关键节点飞书通知HR
```

## 四、触达服务接口定义

### 对表格服务（入口）
| 方法 | 路径 | body | 说明 |
|---|---|---|---|
| POST | `/reach` | `{dataId, phone, interviewTime, name?, position?}` | 发起触达，建任务、调秒回加好友 |

### 给秒懂画布 plugin 调（注册为 PluginApp+Plugin）
| 方法 | 路径 | 参数 | 说明 |
|---|---|---|---|
| GET | `/logic/candidate-info` | `phone` | 画布好友通过后查约面信息发欢迎语 → 返回 `{name, position, interviewTime}` |
| POST | `/logic/report-intent` | `{phone, intent, slots?}` | 画布识别意图后回报 → 触达服务回填+通知HR |

### 秒回回调（配到小组控制台→API接入，单一 URL 分发）
| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/mh/callback` | 统一回调入口，按 body 分发 friend/confirm、friend/send、sentResult、chat |

### 已有（保留）
- `POST /logic/reach` 纯加好友(表格服务也可直接用)、`/logic/notify-hr`、`/logic/progress`、`/feishu/*`、`/admin`

## 五、状态机（Mongo）

```
PENDING → ADDING ─┬─ ADD_FAILED (通知HR, 回填)
                  └─ CONFIRMED(好友通过) → WELCOMED(欢迎语已发) → REPLIED
                        REPLIED ─┬─ INTENT_ACCEPT   (确认约面 → 同步HR+回填)
                                 ├─ INTENT_RESCHEDULE(改约 → 同步HR+回填)
                                 ├─ INTENT_REJECT   (拒绝 → 同步HR+回填, 关闭)
                                 ├─ INTENT_QUESTION (答疑, 不改主状态)
                                 └─ HANDOVER        (转人工)
   超时: WELCOMED 后 N 小时无回复 → REMIND_HR (提醒HR跟进)
```

## 六、数据模型（MongoDB `aihr.reach_tasks`）

```js
{
  _id, taskId,                          // taskId 作 extraInfo 透传
  dataId,                               // 表格记录 id (回填用)
  phone, name, position, interviewTime, // 约面信息
  hrBotUserId,                          // 发起触达的 HR 托管账号
  status,                               // 状态机(见上)
  wxid, externalUserId, chatId,         // 好友通过后填充
  intent, intentSlots,                  // 候选人意图
  timeline: [{ at, event, detail }],    // 全流程留痕
  createdAt, updatedAt,
}
// 幂等: Redis SETNX 锁 phone/messageId/eventId; Mongo taskId 唯一
```
